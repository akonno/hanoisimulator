For Instructors | The Tower of Hanoi Simulator

This simulator runs move commands and visualizes the result; it does not solve the puzzle. Students generate commands with their own programs, and instructors verify correctness visually.

Purpose and classroom usage

This simulator is designed for programming courses where students generate a sequence of move commands for the Tower of Hanoi using their own programs.

The simulator does not solve the puzzle automatically. Instead, it executes student-generated commands and visualizes the result, allowing instructors to verify correctness efficiently.

In typical classroom use, students paste their command output into the simulator, and instructors confirm the result visually (e.g., by checking the final state on screen or a submitted screenshot).

Typical classroom usage

This section describes common instructional patterns when using the simulator in programming courses.

  • Baseline task: Use 7 discs as the standard requirement.
  • Bonus / advanced task: Encourage students to try 8 or more discs for additional credit.
  • Optional challenge: Allow highly motivated students to attempt up to 10 discs, with appropriate time considerations.

Because the required number of moves grows exponentially, simulations with a larger number of discs may take a long time to complete. In one reference environment, transferring 7 discs required over 6 minutes, while 10 discs required more than 50 minutes.

In classroom settings, instructors may verify submissions by checking that the simulation runs without errors and reaches the correct final configuration (often via a screenshot), rather than waiting for the full animation to finish in real time.

Difficulty control (number of discs)

The difficulty of the Tower of Hanoi task can be adjusted by changing the number of discs. Increasing the number of discs significantly increases the number of required moves.

Changing the number of discs

The number of discs can be changed from the Settings menu (top right of the simulator screen). The available range is from 5 to 10 discs, with 7 discs as the default.

This allows instructors to control the task difficulty without changing the assignment itself.

Advanced: Using URL parameters

As an alternative, the number of discs can also be specified using the numDiscs URL parameter. This may be useful when distributing fixed links to students.

  • 5 discs
  • 6 discs
  • 7 discs (default)
  • 8 discs
  • 9 discs
  • 10 discs

The Settings menu is the recommended way to change the number of discs during normal classroom use.

The numDiscs parameter does not impose a strict upper limit. However, because the required number of moves grows exponentially, simulations with values far beyond the intended classroom range will not complete within a realistic amount of time. For example, increasing the disc count by only a few can extend the required execution time from minutes to days or longer.

This behavior can itself serve as a concrete demonstration of exponential growth in algorithmic complexity.
